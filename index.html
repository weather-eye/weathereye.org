<h1><a href="https://www.weathereye.org/">weathereye.org</a></h1>

<h1 id="weathereye">WeatherEye</h1>

<p>WeatherEye is a powerful, integrated suite of free, open-source software created to better support the evolving needs and responsibilities of National Meteorological and Hydrological Services (NMHSs).</p>

<p>The suite consists of the following software (each administered through a tab in the WeatherEye admin interface):</p>
<ol>
<li><strong>DB</strong>: A highly optimised relational database management system (RDBMS)</li>
<li><strong>CDMS</strong>: A system for correctly managing earth system observation data</li>
<li><strong>WIS2</strong>: Software for the discovery and exchange of real-time data using WIS2</li>
<li><strong>API</strong>: Secure, standardised APIs to enable the building of compliant reusable solutions</li>
<li><strong>Portal</strong>: A Content Management System (CMS), fully-integrated to work with the rest of the suite of applications to provide curated access to available data, products and services</li>
</ol>

<p>In addition, the following solutions are planned:</p>
<ol>
<li><strong>Tools ⚒️</strong>: A set of tools, unique to WeatherEye, to better manage the integrated set of applications, including tools to correctly update (migrate) database schemas, backup all data and to secure and audit applications</li>
<li><strong>Desktop</strong>: A forecaster desktop solution</li>
</ol>

<p>WeatherEye builds on existing, high-quality open-source software by integrating these into a single solution with a strong focus on optimising the experience for the user in order to provide accurate and timely weather, climate, and hydrological data management, products and services.</p>

<p>By adopting WeatherEye, NMHSs gain an easier solution for integrating essential software applications and are also guaranteed an easier route to staying up-to-date with WMO advances, including standardisation on first mile data collection, compliance in CDMSs, the standardisation around web APIs and the move to WIS2.</p>

<h2 id="weathereye-components">WeatherEye components</h2>

<p>Each of the free, open-source components are described below.</p>

<h3 id="db">DB</h3>

<p><code class="language-plaintext highlighter-rouge">Status:</code> Available, but not yet integrated<br />
<code class="language-plaintext highlighter-rouge">Powered by:</code> <a href="https://github.com/dbeaver/cloudbeaver">CloudBeaver</a> (Apache 2) + PostgreSQL, TimeScaleDB, PostGIS</p>

<p>All of the applications in WeatherEye are underpinned by a free and open-source enterprise-grade database system built on PostgreSQL with extensions for time series (TimescaleDB) and geospatial (PostGIS) data. WeatherEye brings the management of the data associated with each application into a single RDBMS making it easy to secure, backup and maintain.</p>

<h3 id="cdms">CDMS</h3>

<p><code class="language-plaintext highlighter-rouge">Status:</code> Installable from WeatherEye (SURFACE CDMS install)<br />
<code class="language-plaintext highlighter-rouge">Powered by:</code> <a href="https://github.com/opencdms/surface/">SURFACE CDMS</a> (GPL 3)</p>

<p>WeatherEye uses SURFACE CDMS for it’s Climate Data Management System implementation. SURFACE is a complete CDMS solution that can:</p>
<ul>
<li>Set up multiple user accounts</li>
<li>Be configured for use in a specific country</li>
<li>Manage metadata for station locations, equipment etc.</li>
<li>Ingest data from multiple sources</li>
<li>Perform quality control (QC) on data</li>
<li>Create products</li>
</ul>

<h3 id="wis">WIS</h3>

<p><code class="language-plaintext highlighter-rouge">Status:</code> Available, but not yet integrated<br />
<code class="language-plaintext highlighter-rouge">Powered by:</code> <a href="https://github.com/wmo-im/wis2box">wis2box</a> (Apache 2)</p>

<p>WeatherEye has built-in support to discover, publish and retrieve real-time data from the WIS2 network.</p>

<h2 id="api">API</h2>

<p><code class="language-plaintext highlighter-rouge">Status:</code> In development<br />
<code class="language-plaintext highlighter-rouge">Powered by:</code> <a href="https://github.com/opencdms/opencdms/">OpenCDMS</a> (MIT)</p>

<p>Alongside other applications in the suite, WeatherEye provides a set of interfaces (APIs) that follow the standards being developed by WMO’s Expert Team on Information Management. As time progresses, the other WeatherEye components will increasingly uses these standardised interfaces in their implementation.</p>

<h3 id="portal">Portal</h3>

<p><code class="language-plaintext highlighter-rouge">Status:</code> Available, but not yet integrated<br />
<code class="language-plaintext highlighter-rouge">Powered by:</code> <a href="https://github.com/wmo-raf/nmhs-cms">NMHS-CMS</a> (MIT)</p>

<p>The portal component uses NMHS-CMS, a shared template for the Wagtail (Python/Django) Content Management System (CMS). WeatherEye will further extend NMHS-CMS to make it simple to add reusable “blocks” that are powered by WeatherEye API.</p>

<h3 id="tools-️">Tools ⚒️</h3>

<p><code class="language-plaintext highlighter-rouge">Status:</code> Not yet available<br />
<code class="language-plaintext highlighter-rouge">Powered by:</code> —</p>

<p>The tools component is unique to WeatherEye and will provide migration, backup, security and other additions that may not be present in the individual solutions.</p>

<h3 id="desktop">Desktop</h3>

<p><code class="language-plaintext highlighter-rouge">Status:</code> Not yet available<br />
<code class="language-plaintext highlighter-rouge">Powered by:</code> <a href="https://github.com/geosolutions-it/MapStore2/">MapStore</a> (BSD 2)</p>

<p>A proof-of-concept is in development that uses MapStore as the basis for a web-based forecaster desktop called WeatherEye Desktop. However, this has not yet been made available.</p>

<h2 id="conclusion">Conclusion</h2>

<p>Roadmap and case study sections will be added to this document in the near future giving more details of the software and timeframes until all components will be made fully available.</p>
